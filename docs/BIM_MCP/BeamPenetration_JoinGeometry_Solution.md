# Revit API 穿梁檢核痛點與解決方案：完美捕捉接合面的實體邊界

這份文件記錄了在開發「自動化穿梁套管標註系統」時，為了解決尺寸標註無法精準貼齊梁柱/正交梁邊緣的痛點，所經歷的技術迭代與最終的完美解法。這是一個非常經典且具備高度分享價值的 Revit 二次開發案例！

---

## 💥 痛點描述：消失的截斷面與偏移的中心線

在自動標註套管與相鄰結構（如柱子、正交梁）的淨距時，我們必須精準找到主梁的「物理邊界」。起初，我們面臨了兩個嚴重的技術挑戰：

### 1. Revit 幾何接合 (Join Geometry) 吃掉了平面
一開始，我們嘗試掃描大梁的實體 (Solid)，試圖找出「法向量與梁方向平行 (`dot > 0.8`)」的平面，以此作為梁的端點截斷面。
**但是，這在混凝土結構中完全行不通！**
當大梁與同材質的正交小梁或柱子接合 (Join) 時，Revit 會將它們的幾何體「像黏土一樣融合」。這導致交接處的「平整截斷面」直接從實體拓撲中被刪除消失！程式因為找不到法向量面，最後只好宣告失敗。

### 2. 中心線數學推算法的致命誤差
既然找不到實體面，一般的做法是退而求其次：抓取正交梁的「分析中心線交點」，然後簡單粗暴地「減去半個梁寬 (W / 2)」來當作邊緣。
**但這會產生嚴重的尺寸偏移！**
因為在實際建模中，結構梁經常會有側向對齊 (Justification)、偏移 (Offset)，或者接合時的扣減不均勻。這時候 `中心線 - W/2` 算出來的數學位置，就會跟肉眼看到的真實物理表面產生好幾公分的誤差，導致尺寸線標註不到真正的邊界上，甚至懸空。

---

## 💡 終極解決方案：實體頂點極值投影法 + 1D 拓撲排隊

為了一勞永逸地解決這個因為「3D 幾何融合」帶來的混亂，我們決定**降維打擊**，發明了一套完全不依賴面 (Face) 也不依賴數學推算 (W/2) 的演算法。

### 步驟一：反客為主，直接剖析「障礙物」的實體
既然主梁的面被吃掉了，我們就反過來，直接去讀取那根**「正交梁」或「柱子」本身的實體幾何 (Solid)**。不管它有沒有被接合、有沒有偏移，它在空間中的幾何形狀絕對是最真實的物理存在。

### 步驟二：極值投影 (Min/Max Projection)
我們把這根正交梁實體上**所有的邊緣頂點 (Vertices)**，全部垂直投影到主梁的中心線上。
接著，我們只取兩個極端值：
* **Min 投影點**：代表這根正交梁的「左側物理邊界」。
* **Max 投影點**：代表這根正交梁的「右側物理邊界」。
如此一來，我們成功將一個複雜的 3D 幾何障礙物，轉換成了一段佔據在中心線上的 **「1D 範圍 (Footprint)」**！

### 步驟三：1D 拓撲自動排隊配對
最後，我們把大梁上所有的東西（套管的中心點、大梁的端點、正交梁的投影範圍）全部依照「距離起點的遠近」排成一列。
畫標註線的規則非常簡單：**永遠從「左邊鄰居的右緣」畫到「右邊鄰居的左緣」**。

* **如果套管在正交梁的左邊**：程式會自動從套管畫到正交梁的「左側邊界 (Min)」。
* **如果套管在正交梁的右邊**：程式會自動從正交梁的「右側邊界 (Max)」畫到套管。

## 🏆 結論與價值

這套解法的價值在於：
1. **無視模型不規範**：無論建模人員怎麼偏移、怎麼 Join，只要眼睛看得到的實體邊界，程式就一定抓得到。
2. **跨越連結模型 (Linked Model) 限制**：即使正交梁位在唯讀的連結模型中無法解除接合，我們依然能將其幾何投影過來，完全不受權限影響。
3. **極簡的排隊邏輯**：將複雜的 3D 碰撞問題轉換為 1D 的線段排隊問題，不僅執行效率高，而且絕對不會出現「標到物體另一面」的低級錯誤。

這個「實體頂點極值投影法」徹底解決了 Revit API 在處理自動化標註時最令人頭痛的幾何邊界捕捉問題！
